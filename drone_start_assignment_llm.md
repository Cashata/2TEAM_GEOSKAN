# Задание «Дроно-старт»

## 1. Краткое описание

Команде необходимо разработать систему автономного поиска и доставки целей тремя мини-БВП на арене размером **3 × 3 м**.

На полу арены размещается заранее подготовленная карта района Санкт-Петербурга. На карте отмечены:

- зона старта и посадки;
- база для доставки найденных целей;
- запретная зона — условная школа в центральной части карты;
- зоны возможного размещения целей;
- несколько ArUco-маркеров разных размеров.

Дроны должны автономно ориентироваться по карте, искать маркеры, определять их тип и выполнять действия в зависимости от результата распознавания.

## 2. Цель миссии

Система должна:

1. Запустить три дрона из зоны взлета.
2. Определить положение каждого дрона на карте по камере.
3. Построить маршруты с учетом запретной зоны.
4. Обследовать зоны возможного размещения целей.
5. Найти ArUco-маркеры.
6. Определить ID каждого найденного маркера и его координаты.
7. Сверить ID со словарем ArUco-маркеров.
8. Определить тип цели:
   - разрешенная цель;
   - запрещенная цель.
9. Для разрешенных целей:
   - захватить цель магнитным захватом;
   - доставить цель на базу.
10. Для запрещенных целей:
    - отсканировать цель;
    - зафиксировать ID;
    - передать информацию в систему;
    - не перевозить цель.
11. Собрать слово или короткую фразу из букв, соответствующих найденным ArUco-маркерам.
12. Завершить миссию после выполнения условий завершения.

## 3. Состав команды и роли

Команда должна распределить работу между участниками. В задании есть три основных направления.

### 3.1. Программная часть

Участники должны реализовать:

- определение положения дрона на карте;
- распознавание ArUco-маркеров;
- построение маршрута с учетом запретных зон;
- координацию между несколькими дронами;
- выбор стратегии поиска;
- возврат на базу;
- обработку найденных разрешенных и запрещенных целей.

### 3.2. Конструкторская часть

Конструктор команды должен разработать и изготовить крепление для магнитного захвата.

Требования к креплению:

- крепеж устанавливается на дрон;
- предусмотрено место для магнита или магнитного модуля;
- крепление должно быть достаточно легким, чтобы не мешать полету;
- крепление должно обеспечивать надежное удержание цели при переносе;
- крепление не должно перекрывать камеру, датчики и винты дрона;
- крепление не должно нарушать балансировку дрона;
- крепление должно обеспечивать возможность быстрого монтажа и демонтажа.

Цель с ArUco-маркером будет иметь ответную магнитную часть, чтобы дрон мог захватить ее и перенести на базу.

### 3.3. Интеграционная часть

Команда должна объединить:

- навигацию;
- компьютерное зрение;
- управление полетом;
- мультиагентное взаимодействие;
- магнитный захват;
- доставку цели на базу.

## 4. Арена

| Параметр | Значение |
|---|---|
| Форма арены | квадрат |
| Размер арены | 3 × 3 м |
| Основа арены | карта района Санкт-Петербурга |
| Обязательные зоны | зона старта-посадки, база, запретная зона, зона поиска |
| Объекты на карте | ArUco-маркеры разных размеров |

На полу размещена карта района. На карте заранее нанесены или заданы организаторами:

- зона старта-посадки;
- запретная зона;
- зона поиска;
- ArUco-маркеры.

## 5. Цели

На арене размещены **4 ArUco-маркера разных размеров**.

Маркеры делятся на два типа.

### 5.1. Разрешенные цели

Разрешенные цели нужно:

1. Найти.
2. Распознать.
3. Захватить магнитным захватом.
4. Доставить на базу.

### 5.2. Запрещенные цели

Запрещенные цели нужно:

1. Найти.
2. Отсканировать.
3. Определить ID.
4. Передать информацию в систему.

Запрещенные цели **нельзя перевозить**.

## 6. Словарь ArUco-маркеров

Перед запуском команде выдается словарь ArUco-маркеров.

В словаре указано:

- какие ID являются разрешенными целями;
- какие ID являются запрещенными целями;
- какая буква соответствует каждому ID.

### 6.1. Словарь из задания

| ID маркера | Тип цели | Буква | Действие |
|---:|---|---|---|
| 3 | разрешенная цель | И | найти, распознать, захватить, доставить на базу |
| 23 | разрешенная цель | Т | найти, распознать, захватить, доставить на базу |
| 42 | запрещенная цель | М | найти, отсканировать, зафиксировать ID, не перевозить |
| 117 | запрещенная цель | О | найти, отсканировать, зафиксировать ID, не перевозить |

### 6.2. Восстановление слова или фразы

Команда должна определить ID найденных маркеров и восстановить слово или короткую фразу из букв, соответствующих найденным ArUco-маркерам.

Правило восстановления:

- порядок букв определяется по возрастанию ID маркеров;
- формат словаря может быть изменен организаторами.

Для словаря из задания порядок по возрастанию ID:

1. ID 3 → «И»;
2. ID 23 → «Т»;
3. ID 42 → «М»;
4. ID 117 → «О».

Итоговое слово для текущего словаря: **«ИТМО»**.

## 7. Формат выступления и попыток

Перед запуском команда кратко представляет свое решение.

Время на представление: **до 5 минут**.

Команда должна рассказать:

- как устроен алгоритм поиска;
- как выполняется распознавание ArUco-маркеров;
- как дроны координируются между собой;
- как устроено магнитное крепление;
- как система определяет разрешенные и запрещенные цели;
- как собирается слово или фраза по словарю ArUco-маркеров.

После представления команда выполняет полетные попытки.

Количество попыток для каждой команды: **2 попытки**.

## 8. Сценарий попытки

1. Дроны стартуют из зоны взлета.
2. Дроны определяют свое положение на карте по камере.
3. Система строит маршрут с обходом запретной зоны.
4. Дроны обследуют зоны возможного размещения целей.
5. При обнаружении ArUco-маркера дрон определяет:
   - ID маркера;
   - координаты маркера.
6. Система сверяет ID со словарем.
7. Система определяет соответствующую букву.
8. Дрон передает информацию остальным дронам или центральной программе.
9. По итогам поиска команда собирает слово или короткую фразу из букв найденных маркеров.
10. Если цель разрешенная:
    - дрон захватывает цель магнитным креплением;
    - дрон доставляет цель на базу.
11. Если цель запрещенная:
    - дрон только сканирует цель;
    - система фиксирует ID цели;
    - цель не перевозится.
12. Миссия завершается при выполнении одного из условий завершения.

## 9. Условия завершения миссии

Миссия завершается, когда выполнено хотя бы одно из условий:

- найдены все маркеры и доставлены все разрешенные цели;
- истекло время попытки;
- команда завершила выполнение вручную.

## 10. Результат попытки

После завершения миссии команда должна показать:

- список найденных ID;
- координаты найденных маркеров;
- какие цели были разрешенными и запрещенными;
- буквы, соответствующие найденным ArUco-маркерам;
- собранное слово или короткую фразу;
- какие разрешенные цели доставлены на базу;
- краткое объяснение алгоритма поиска и конструкции магнитного крепления.
